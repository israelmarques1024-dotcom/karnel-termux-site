Why Omni Catalyst?

Everything you need to code on your Android device, packaged in a modular framework with native tooling

AI

26 AI Coding Agents

OpenCode, Claude Code, Gemini CLI, Codex, Freebuff, Qwen, MiMo, Kimi, Kiro, and more — pre-configured for Termux. Install individually or batch install all.

MF

Modular Architecture

Install only what you need. Each tool has its own installer with install, uninstall, update, and reinstall commands. CLI flags or interactive TUI menu.

NE

Native Execution

Automatic glibc setup for native binaries via glibc-prefix. Falls back to proot-distro Ubuntu containers when native execution is not available.

BR

Second Brain

Built-in memory system with AI-powered search, semantic embeddings, graph visualization, and automatic GitHub synchronization. Save and retrieve context across sessions.

UI

Terminal Experience

Interactive TUI menus with fzf, real-time progress spinners with gum / pv, startup banner with gradient palette, and carefully designed color schemes.

DX

Doctor Diagnostics

Comprehensive environment audit with auto-correction. Scans 20+ categories — PATH, shell config, language runtimes, databases, AI tools, git setup, and more — then fixes issues automatically.

DP

Deploy CLIs

Vercel, Railway, and Netlify CLIs pre-configured. Deploy your projects directly from Termux without a separate computer. Full CI/CD pipeline from your phone.

NP

npm Publishing

Published as omni on npm. Install globally with npm install -g omni. Full npm ecosystem integration for package management and script running.

Termux Puro vs Omni Catalyst

Why a modular framework transforms your Termux experience

✕ Manual Termux

  • ✕ Each tool installed manually via pkg install — no central management
  • ✕ Environment variables spread across .bashrc, .zshrc, .profile — no centralized control
  • ✕ No module system — uninstalling leaves orphan configs and dependencies
  • ✕ No diagnostics — broken configs or conflicts are hard to detect
  • ✕ No terminal banner, visual identity, or startup customization
  • ✕ AI agents require manual installation of Python/npm packages and API keys
  • ✕ No memory system — context lost between terminal sessions
  • ✕ Native glibc binaries (Codex, Claude, etc.) fail without manual glibc setup
  • ✕ Deploy CLIs (Vercel, Railway) installed and updated individually
  • ✕ Database init scripts, config files, and systemd services all manual

✓ Omni Catalyst

  • ✓ Single-command install — any tool with omni install <module> --tool
  • ✓ Centralized env management via omni env set — one source of truth
  • ✓ Modular install/uninstall/update/reinstall — clean removal guaranteed
  • ✓ omni doctor — audits 20+ categories with auto-correction
  • ✓ Customizable startup banner with TrueColor gradient and cyberpunk aesthetic
  • ✓ 26 AI agents pre-configured — single command install, ready to run
  • ✓ Second Brain — semantic search, graph viz, GitHub sync
  • ✓ Automatic glibc-prefix + proot-distro fallback for any binary
  • ✓ Vercel, Railway, Netlify — install once, update via omni update
  • ✓ PostgreSQL init/start/shell via omni pg — no manual config

Step-by-Step Guide

# 1. Install Termux + Termux:API from GitHub or F-Droid
#    Download links: https://github.com/termux/termux-app/releases
#                   https://github.com/termux/termux-api-package/releases

# 2. Open Termux and update packages
$ pkg update && pkg upgrade -y

# 3. Install curl if not present
$ pkg install curl -y

# 4. Run the Omni installer
$ bash -c "$(curl -fsSL https://raw.githubusercontent.com/israel676767/omni/main/install.sh)"

# 5. Verify installation
$ omni --version

# 6. Install what you need
$ omni install ai --opencode    # Install an AI agent
$ omni install lang             # Install all languages
$ omni install shell            # Install ZSH + Oh My Zsh

Post-Installation

After installation, run these commands to explore:

# List all available tools in a module
$ omni list ai

# Check system health
$ omni doctor

# Open documentation website
$ omni open omni

# See all commands
$ omni --help
